🔗 Collection Relationships
organizations → buildings
One-to-Many: One organization can have multiple buildings.
buildings.organizationId → organizations.id
buildings → floors
One-to-Many: One building can have multiple floors.
floors.buildingId → buildings.id
floors → rooms
One-to-Many: One floor can have multiple rooms.
rooms.floorId → floors.id
floors → corridors
One-to-Many: One floor can have multiple corridors.
corridors.floorId → floors.id
rooms ↔ corridors
Many-to-Many: Corridors connect rooms together.
corridors.startRoomId → rooms.id
corridors.endRoomId → rooms.id
users (standalone)
Independent: User authentication and role management.

📝 Implementation Notes:
  • Firestore Structure: All collections are top-level (not nested) for better query performance
  • Graph Construction: Rooms and corridors form a navigation graph for A* pathfinding
  • Accessibility: isAccessible flags enable wheelchair-friendly routing
  • Coordinates: x, y positions are relative to floor map images
  • Timestamps: All entities track creation/update times for audit trails
