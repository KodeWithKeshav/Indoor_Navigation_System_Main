EPIC 1: Indoor Map Data Setup & Management
User Story 1.1

As a system administrator, I want to add buildings to digitally represent campus structure.

1.1.1 Design the building data schema with unique identifiers.
1.1.2 Create an interface to input building details.
1.1.3 Validate and store building data in the system.

User Story 1.2

As a system administrator, I want to define floors to support multi-floor navigation.

1.2.1 Link floors to their corresponding building IDs.
1.2.2 Validate floor numbering and hierarchy rules.
1.2.3 Store floor data and update building mappings.

User Story 1.3

As a system administrator, I want to add rooms for precise indoor location selection.

1.3.1 Generate unique room IDs and assign them to floors.
1.3.2 Validate room uniqueness within each floor.
1.3.3 Save room metadata to the location database.

User Story 1.4

As a system administrator, I want to define corridors to establish indoor connectivity.

1.4.1 Identify rooms that should be connected by corridors.
1.4.2 Create bidirectional corridor links between rooms.
1.4.3 Update the navigation graph with corridor paths.

User Story 1.5

As a system administrator, I want to define stairs for vertical floor movement.

1.5.1 Identify stair locations on each relevant floor.
1.5.2 Link stair nodes across multiple floors.
1.5.3 Store stair connections in the navigation graph.

User Story 1.6

As a system administrator, I want to define elevators for accessible vertical navigation.

1.6.1 Identify elevator nodes within buildings.
1.6.2 Connect elevator nodes to supported floors.
1.6.3 Store elevator data with accessibility attributes.

User Story 1.7

As a system administrator, I want to tag accessibility features to enable inclusive routing.

1.7.1 Define accessibility attributes for map entities.
1.7.2 Attach accessibility tags to rooms and paths.
1.7.3 Save accessibility metadata for routing logic.

User Story 1.8

As a system administrator, I want to edit map data to reflect layout changes.

1.8.1 Enable selection and modification of map entities.
1.8.2 Revalidate connectivity after data changes.
1.8.3 Save updated map data to the system.

User Story 1.9

As a system administrator, I want to remove outdated locations to maintain map accuracy.

1.9.1 Identify outdated or unused map entities.
1.9.2 Remove associated links while preserving consistency.
1.9.3 Update the navigation graph after deletion.


EPIC 2: User Input for Location & Destination
User Story 2.1 As a user, I want to select my starting building and room for navigation.

2.1.1 Load and display a list of available buildings.

2.1.2 Filter and display rooms based on the selected building.

2.1.3 Capture and validate the user's starting coordinates.

User Story 2.2 As a user, I want to select my destination building and room to plan routes.

2.2.1 Provide a selection interface for destination buildings.

2.2.2 Filter destination rooms to ensure they are reachable from the starting point.

2.2.3 Store the destination ID for route calculation.

User Story 2.3 As a user, I want a search bar to find specific locations by name quickly.

2.3.1 Implement a real-time search algorithm for rooms and points of interest.

2.3.2 Filter and display matching results as the user types.

2.3.3 Allow direct selection from the search results to populate the destination field.

User Story 2.4 As a user, I want locations grouped by floor to browse the campus hierarchy easily.

2.4.1 Organize map entities into a nested data structure (Building > Floor > Room).

2.4.2 Design a grouped UI components to represent the campus hierarchy.

2.4.3 Implement "accordion" or "drill-down" navigation for location selection.

User Story 2.5 As a user, I want to view recent locations to repeat frequent navigation paths faster.

2.5.1 Cache the user's most recently selected starting points and destinations.

2.5.2 Display a "Recent" or "Favorites" shortcut list on the main input screen.

2.5.3 Manage the history limit to ensure only relevant, recent data is shown.

User Story 2.6 As a user, I want to swap my start and end points for return trips.

2.6.1 Create a "Swap" toggle button between the start and end input fields.

2.6.2 Update the navigation logic to re-calculate the path in reverse.

2.6.3 Re-validate the new path for one-way constraints or accessibility.

User Story 2.7 As a user, I want the system to block invalid selections to avoid routing errors.

2.7.1 Implement real-time validation to block "Start" and "End" being the same location.

2.7.2 Flag and hide rooms currently marked as "Out of Service" or "Closed."

2.7.3 Display clear error messages when an invalid location combination is attempted.

User Story 2.8 As a user, I want to edit my selected locations to correct any input mistakes.

2.8.1 Enable "Clear" or "Edit" buttons on selected location tags.

2.8.2 Reset the dependent UI states when a building selection is changed.

2.8.3 Ensure the navigation graph clears previous path overlays upon modification.

User Story 2.9 As a user, I want a trip summary before generating the final navigation path.

2.9.1 Display a summary card showing the selected Start, End, and total floor distance.

2.9.2 Provide a final "Get Directions" trigger to initiate the routing engine.

2.9.3 Capture user confirmation to begin the turn-by-turn navigation process.
